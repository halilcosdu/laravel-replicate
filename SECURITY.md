# Security Policy

## Supported Versions

Security fixes are provided for the latest minor release. Upgrade before reporting an issue that only affects an older line.

| Version | Supported |
| --- | --- |
| 2.x | ✅ |
| 1.x | ❌ |

## Reporting a Vulnerability

Please report vulnerabilities through a [private GitHub security advisory](https://github.com/halilcosdu/laravel-replicate/security/advisories/new). If that is not possible, email halilcosdu@gmail.com with a description, reproduction steps, affected versions, and any known mitigation.

Do not open a public issue or disclose the vulnerability until a fix is available. You should receive an acknowledgement as soon as the report has been reviewed.

## Security Updates

Fixes are published as tagged releases and documented in the changelog. Always use the latest stable package version.

## More Information

For non-sensitive questions, use the repository's public issue tracker.
